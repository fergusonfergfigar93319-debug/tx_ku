# 元流同频 — Web 端改版对齐说明

**版本**：V1.0  
**读者**：Web 前端、全栈、产品、测试  
**目的**：在 **不推翻《同频搭_Web端详细开发方案》《Web前端技术栈选型-Vue3》技术选型** 的前提下，明确 **Web 端须按 Android 当前版本做一次「大调整」**：产品名、信息架构、模块边界、枚举与文案与 App **同构**，并与 **`TX_ku_Android前后端对接说明.md`** 共用接口与数据契约。  
**Android 对照**：`MainTabScreen.kt`（底栏四 Tab）、`Routes.kt`、`BrandConfig.kt`、`AgentTuningOptions`、`FollowGameCatalog`、`ForumScreen` / `PostListScreen`、`ProfileScreen`、`AgentPersonaScreen`。

---

## 一、为何要做这次改版

早期 Web 方案以 **「同频搭 / BuddyCard / 发现·搭子·广场·我的」** 为叙事，且隐含 **多游戏、泛品类** 扩展。当前 Android **已收口为《王者荣耀》+ 王者电竞向社区工具**，应用显示名为 **元流同频**，底栏与站内文案已全面替换。若 Web 仍沿用旧命名与旧信息架构，将出现：

- 用户从 App 跳到 H5 **认知不一致**（找不到「版本速递」「元流档案」「峡谷广场」）。
- 智能体 **`focusScenario` / `avatarStyle`** 与后端校验枚举 **与 App 不一致**，联调失败。
- 游戏兴趣若仍展示三角洲、CS 等 **App 已不提供的选项**，属于产品回归错误。

因此：**Web 端应以 App 现版为单一真相源（SSOT），本说明为改版需求文档；具体组件实现仍可按 Vue3 + Element Plus 原方案推进。**

---

## 二、大调整总览（对照旧 Web 方案）

| 维度 | 旧 Web 方案（或常见误解） | **现版要求（与 App 一致）** |
|------|---------------------------|----------------------------|
| 产品名 | 同频搭 | **元流同频**（全站主品牌；页面 `<title>`、关于、分享、运营后台展示统一） |
| 底栏 Tab 1 | 「发现」 | **版本速递**（路径可仍用 `/app/feed` 等，**展示文案必须为「版本速递」**） |
| 底栏 Tab 2 | 「搭子」 | **AI搭子** |
| 底栏 Tab 3 | 「广场」 | **峡谷广场**（列表顶区、浏览器标题后缀可与 App 一致） |
| 底栏 Tab 4 | 「我的」 | **元流档案** |
| 游戏兴趣 | 米游社式多品类示例 | **仅两项**：`王者荣耀`、`王者电竞`（与 `FollowGameCatalog.options` 的 `id` 一致） |
| 智能体场景 | 可含三角洲、CS:GO 等 | **仅**：`通用`、`组队招募`、`赛后复盘`、`缓解压力`、`王者荣耀`、`王者电竞` |
| 成品智能体 | 多游戏壳 | **以王者高热度英雄主题 + 分路 / 赛事实况 / 复盘** 为主（概念对齐 `DesignedAgentPresets`，无需像素级复刻立绘） |
| 资讯流「官方」发布者 | 未强调 | 展示名对齐 **`元流同频官方`**（产品内栏目，非厂商本体；与 `BrandConfig.officialPublisherName` 一致） |
| 论坛心智 | 泛「论坛」 | **峡谷广场**；发帖页、空状态、审核提示用语与 App 对齐 |
| 个人中心 | 泛「我的」 | **元流档案**；「我的帖子」入口文案建议：**峡谷广场 · 我的帖子**（与 App sheet 标题一致） |

## 三、信息架构与路由（建议）

在保留 **Vue Router 4 + 主壳布局** 的前提下，建议 **路径稳定、仅改导航标签与面包屑**，URL 不必强行中文。

| App `MainTab` | Web 建议 path（示例） | 导航栏/底栏 **展示文案** |
|---------------|----------------------|-------------------------|
| FEED | `/app/feed`（默认子路由） | **版本速递** |
| AGENT | `/app/agent` | **AI搭子** |
| FORUM | `/app/plaza` 或 `/app/forum` | **峡谷广场** |
| PROFILE | `/app/archive` 或 `/app/me` | **元流档案** |

**说明**：

- 若历史代码已使用 `/app/discover`、`/app/me`，可 **保留 path 做重定向**（`discover` → `feed`），但 **任何用户可见标签不得再写「发现」「我的」作为主 Tab 名**。
- **全屏页**与 Android `Routes.kt` 对齐：`/login`、`/register`、`/onboarding`、`/game-interest`、`/app/.../post/:postId`、`/post-editor`、`/app/agent/chat`、`/app/archive/profile-edit`、`following`、`add-friend`、`dm/:peerUserId` 等（具体 path 团队可微调，但 **守卫逻辑与 Android 一致**：未登录、未建档、未完成游戏兴趣、未解锁聊天进会话页等）。

## 四、分模块改版要点

### 4.1 版本速递（原「发现」）

- **结构**：**资讯子 Tab** + **推荐搭子子 Tab**，与 App `FeedScreen` 一致。
- **资讯**：频道与 **游戏兴趣** 联动仍是 **`王者荣耀` / `王者电竞`**；接口见对接说明 `GET /feed/news`（若上收）。
- **文案**：首页顶栏、侧栏、HTML title 使用 **元流同频**，副线使用「版本速递」而非「发现」。
- **场景捷径/跳转**：从首页进入搜索、分区、发帖、AI 搭子预填等，**目标 Tab 名称**在提示里写 **峡谷广场** / **AI搭子** / **元流档案**，与 App 内 snackbar 一致。

### 4.2 AI搭子（智能体创作 + 聊天）

- **表单模型**：与 Android `AgentTuning` **字段名、可选值逐字一致**（见 `AgentTuningOptions`）。
- **`focusScenario`**：`通用`、`组队招募`、`赛后复盘`、`缓解压力`、`王者荣耀`、`王者电竞`。**禁止**向用户展示已删除的「三角洲行动」「CS:GO」等（后端若存旧值需迁移或拒绝写入）。
- **`avatarStyle`**：与对接说明及 `AgentTuningOptions.avatarStyles` 一致（含 `英雄主题·*`、`赛事实况台`、分路/军师类兼容键）。Web 可 **用缩略图或纯色占位** 映射，不必接入 App 同资源 id，但 **提交给后端的字符串必须同源**。
- **成品一键套用**：产品策略对齐 Android「王者英雄壳 + 打法话术」；Web 可 **精简卡片数量**，但 **预设写入的 `AgentTuning` 字段集合**应与安卓同类预设等价（或由服务端下发同一 JSON 模板）。
- **解锁聊天**：`agentChatUnlocked` 行为与守卫与 App 一致；未解锁访问 `/app/agent/chat` → 引导回 **AI搭子** 完成创作。

### 4.3 峡谷广场（论坛）

- **列表/搜索/分区**：`categoryId`、`Post`、`PostModerationStatus`、`isVisibleInPublicForum()` 语义与 **`TX_ku_Android前后端对接说明.md` §2.7** 完全一致。
- **顶区文案**：页面主标题 / H1 级语义：**峡谷广场**；副标题可对应「开黑招募 · 攻略 · 赛评唠局」。
- **发帖流程**：标题 **发到峡谷广场** 类文案、审核中提示与会话 **对齐 `PostEditorScreen` / `strings.xml`** 的精神（不必逐字拷贝，但不可再写「仅需发广场」而忽略审核）。
- **详情评论**：仅 `APPROVED` 且对公域可见时开放评论（与 App 一致）。

### 4.4 元流档案（个人中心）

- **顶区品牌**：主标题 **元流档案**；副文案可呼应「搭子人设 · 资料进度 · 与峡谷广场同频」等 App 口径。
- **功能入口**：资料编辑、关注、加好友、收藏、**峡谷广场 · 我的帖子**、组队名片（`BuddyCard`）、退出登录；与 `ProfileScreen` 能力对齐。
- **「我的帖子」**：作者全状态列表（含待审）建议对接 `GET /users/me/posts`；公域列表仍只展示已过审帖。

### 4.5 游戏兴趣页

- **选项固定为两条**：`王者荣耀`、`王者电竞`（`id` 与 Android `FollowGameOption.id` 相同）。
- **完成标记**：未完成则拦截进主壳（与 App `GameInterestStore` 行为一致；最终以服务端 `GET /profiles/me` 或专用接口为准）。

### 4.6 认证、建档、社交、搭子房

- **无结构性变更** relative to《同频搭_Web端详细开发方案》，但所有 **用户可见「产品名」「Tab 名」「论坛名」** 按本文件第二节替换。
- **搭子房 / 共识卡**：仍为 Phase 2+；路由可保留，**不在首版阻断「四 Tab 对齐」验收**。

## 五、品牌、合规与复制文案

- **应用名**：**元流同频**（Web 页脚/关于/分享卡片第一行）。
- **非官方声明**：建议在注册页脚、关于页、发帖提示等处保留与 Android `store_listing_full` **同等效力的说明**（本应用为第三方工具，非《王者荣耀》客户端或腾讯官方产品等）。
- **智能体分享/剪贴板**：若 Web 也做「复制人设卡/档案摘要」，标题行对齐  
  `【元流同频 · 人设卡】` / `【元流同频 · 档案摘要】`（见 `BrandConfig`）。

## 六、设计系统（Web 侧）

- **色板方向**：与 Android `BuddyColors` 一致 — **电竞紫 + 赛博青 + 社区蓝**；论坛浅色区可偏 **天青灰底**，与 App `ForumPlazaTheme` / 资讯区 **同系**，避免旧方案里与「薄荷绿广场」冲突的独立色（若旧稿仍写青绿条，改为 **紫青渐变强调条**）。
- **元流档案顶区**：可有 **浅紫 + 社区蓝渐变的 hero 横幅**（对齐 App `ProfileHeaderBanner` 思路）。
- **底栏**：四 Tab **文案字数较多**（如「峡谷广场」「元流档案」），Web 大屏可用 **侧栏 + 顶栏**；移动端 H5 则需 **缩小字号 / 省略号**，与 Android `labelSmall` + `maxLines=1` 策略同思路。
- **无障碍与对比度**：维持原 Web 方案 §七、§八 要求。

## 七、API 与类型

- **唯一契约**：以 **`TX_ku_Android前后端对接说明.md`** 与 **`同频搭_API与数据定义文档.md`**（文首已标注元流同频）为准；Web **TypeScript interface** 字段名与 Android `core.model` **camelCase 对齐**。
- **枚举**：`PostModerationStatus`、`categoryId`、`focusScenario`、`avatarStyle` 等 **禁止前端自创别名**；与后端书面锁定一份「允许取值清单」。

---

## 八、推荐实施顺序（改版任务拆分）

1. **全局替换**：产品名、Tab 文案、路由守卫提示、404/空状态中的旧「发现」「我的」「广场」。
2. **游戏兴趣**：选项裁切为两项 + 联调 `gameIds`。
3. **智能体表单**：`AgentTuning` 选项与成品预设与 App 对齐；联调 PUT 画像扩展字段。
4. **峡谷广场**：列表顶区与审核逻辑、我的帖子作者视图。
5. **元流档案**：布局与文案、帖子抽屉/独立页。
6. **回归**：对照 **第二节表格** 做验收勾选。

## 九、验收检查清单（节选）

- [ ] 论坛公域列表仅展示审核通过帖；作者可在元流档案侧看到待审帖。  
- [ ] 页脚或关于含 **非官方第三方** 合规说明。
